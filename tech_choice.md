| Component                 | Selected Option                                          | Why This Was Chosen                                                                                                                                      | Alternatives Considered            | Why Not the Alternatives                                                                                                                                              |
| ------------------------- | -------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Dataset                   | **KITTI Raw**                                            | Standard dataset for robotics and autonomous driving. Provides diverse real-world driving scenes useful for learning scene-level visual representations. | TUM RGB-D                          | TUM focuses on indoor scenes and depth-based tasks. KITTI better represents real-world robotics perception scenarios and has more diverse outdoor visual features.    |
| Learning Paradigm         | **Self-Supervised Learning**                             | Allows the model to learn visual representations without labels, which mirrors real-world robotics data collection where unlabeled data is abundant.     | Fully supervised learning          | Requires large labeled datasets. The assignment specifically tests whether useful representations can be learned **without labels**.                                  |
| SSL Method                | **SimCLR**                                               | Conceptually simple, widely studied, strong performance in representation learning, and easy to implement with limited compute.                          | BYOL, MoCo                         | BYOL requires momentum networks and more complex training dynamics. MoCo requires memory banks and queues, making implementation more complex for a small assignment. |
| Backbone Network          | **ResNet-18**                                            | Lightweight CNN with residual connections that stabilize training. Suitable for small datasets and fast experimentation.                                 | ResNet50, Vision Transformers      | Larger models require more data and compute, increasing risk of overfitting on a 500-image dataset.                                                                   |
| Training Dataset Size     | **≤500 Unlabeled Frames**                                | Matches assignment constraints while keeping training manageable. Enough variation to learn useful representations.                                      | Full dataset training              | Computationally expensive and unnecessary for demonstrating representation learning concepts.                                                                         |
| Downstream Task           | **Small Scene Classification Task**                      | Simple way to evaluate whether learned features capture semantic information about environments.                                                         | Object detection, segmentation     | These tasks require more labels and complex pipelines which are outside the scope of this assignment.                                                                 |
| Evaluation Strategy       | **Pretrained vs Training from Scratch**                  | Provides a controlled experiment to measure the benefit of representation learning.                                                                      | Only evaluating pretrained model   | Without a scratch baseline, it is impossible to quantify the actual benefit of pretraining.                                                                           |
| Visualization Method 1    | **PCA**                                                  | Captures global variance structure in the embeddings and provides interpretable dimensionality reduction.                                                | Only t-SNE                         | PCA helps understand global feature structure which t-SNE does not preserve.                                                                                          |
| Visualization Method 2    | **t-SNE**                                                | Excellent at revealing local clusters and semantic grouping in high-dimensional embeddings.                                                              | UMAP                               | UMAP is good but t-SNE is more widely recognized and expected in research-style representation learning analysis.                                                     |
| Clustering Evaluation     | **Silhouette Score**                                     | Measures how well embeddings form compact and well-separated clusters.                                                                                   | No quantitative cluster evaluation | Visualization alone can be subjective. Silhouette score provides objective evaluation.                                                                                |
| Cluster Separation Metric | **Davies–Bouldin Index**                                 | Measures similarity between clusters and penalizes overlapping clusters.                                                                                 | Only one clustering metric         | Using multiple metrics strengthens evaluation and increases reliability of conclusions.                                                                               |
| Data Augmentation         | **Contrastive Augmentations (crop, color jitter, blur)** | Augmentations force the model to learn invariant representations rather than memorizing pixel patterns.                                                  | Minimal augmentation               | Without augmentation, contrastive learning fails because positive pairs become too similar.                                                                           |
| Embedding Source          | **Encoder features before projection head**              | The encoder output contains generalizable representations used for downstream tasks.                                                                     | Using projection head embeddings   | Projection head embeddings are optimized only for contrastive loss and may not generalize as well for downstream tasks.                                               |
